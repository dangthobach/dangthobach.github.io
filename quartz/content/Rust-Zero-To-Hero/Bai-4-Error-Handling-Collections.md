# Bài 4: Error Handling & Collections - Tạm biệt Null và Try-Catch

Chào Chuyên gia Java, trong thế giới Spring Boot, chúng ta đã quá quen với việc ném `RuntimeException` hoặc kiểm tra `if (obj != null)`. Trong Rust, chúng ta xử lý lỗi và dữ liệu tập hợp (Collections) theo một cách "tường minh" hơn nhiều, giúp loại bỏ hoàn toàn lỗi runtime kinh điển.

## 1. Error Handling: Không có Exception, chỉ có Enum

Rust không có `try-catch`. Thay vào đó, nó sử dụng hai Enum cực kỳ mạnh mẽ: `Option<T>` và `Result<T, E>`.

### Option<T>: Thay thế cho Null
Trong Java, một phương thức trả về `String` có thể là "Hello" hoặc `null`. Trong Rust, bạn phải khai báo rõ:

```rust
fn find_user_name(id: u32) -> Option<String> {
    if id == 1 {
        Some(String::from("Alice"))
    } else {
        None // Thay cho null
    }
}
```
**Tại sao Java Dev sẽ thích?** Bạn không thể quên kiểm tra `null`. Compiler sẽ bắt bạn phải "mở hộp" (unwrap/match) cái `Option` đó mới lấy được dữ liệu bên trong.

### Result<T, E>: Thay thế cho Exceptions
Thay vì ném lỗi lên tầng trên và hy vọng có ai đó catch, Rust trả về một kết quả:

```rust
fn divide(a: f64, b: f64) -> Result<f64, String> {
    if b == 0.0 {
        Err(String::from("Không thể chia cho 0"))
    } else {
        Ok(a / b)
    }
}
```

## 2. Toán tử `?` - "Cái bẫy" ngọt ngào của Rust

Trong Java, code của bạn thường bị "phình" to bởi các khối `try-catch`. Trong Rust, chúng ta dùng toán tử `?` để truyền lỗi (error propagation) cực kỳ gọn nhẹ.

```rust
fn process_data() -> Result<(), MyError> {
    let data = fetch_from_db()?; // Nếu lỗi, trả về Err ngay lập tức
    let parsed = parse_json(data)?; // Nếu lỗi, trả về Err
    Ok(())
}
```
Nó tương đương với việc bạn khai báo `throws Exception` trong Java nhưng an toàn và tường minh hơn.

## 3. Common Collections: Bộ ba nguyên tử

### Vector (`Vec<T>`)
Tương đương với `ArrayList<T>` trong Java. Dữ liệu nằm trên Heap và có thể co giãn.
```rust
let mut v = vec![1, 2, 3];
v.push(4);
```

### HashMap (`HashMap<K, V>`)
Tương đương với `HashMap<K, V>` trong Java. 
```rust
use std::collections::HashMap;
let mut scores = HashMap::new();
scores.insert(String::from("Blue"), 10);
```

### String vs &str (Nỗi đau của người mới)
Trong Java chỉ có `String`. Trong Rust có 2 loại chính:
*   `String`: Kiểu sở hữu (Owned), có thể thay đổi kích thước (giống `StringBuilder` nhưng dùng như `String`).
*   `&str`: Kiểu mượn (String slice), là một con trỏ trỏ vào một vùng nhớ chuỗi có sẵn.

## 4. So sánh Tư duy (Cheat Sheet)

| Tính năng | Java / Spring Boot | Rust |
| :--- | :--- | :--- |
| Lỗi không mong muốn | `NullPointerException` | `Option<T> (None)` |
| Xử lý lỗi | `try { ... } catch` | `match` hoặc toán tử `?` |
| Danh sách | `ArrayList<T>` | `Vec<T>` |
| Bản đồ | `HashMap<K, V>` | `HashMap<K, V>` |
| Chuỗi ký tự | `String` | `String` (Heap) & `&str` (Slice) |
