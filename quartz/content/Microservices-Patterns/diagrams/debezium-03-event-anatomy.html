Anatomy của một Debezium Change Event

Click vào từng block để xem chi tiết. Chọn operation type để thấy before/after thay đổi.

Op type:
"before"
null
↓ click để xem
"after"
{ id:1, status:"APPROVED" }
↓ click để xem
"op"
"c"
"source"
lsn, txId, ts_ms...
↓ click để xem
before — Row state TRƯỚC khi thay đổi
after — Row state SAU khi thay đổi
op — Operation type
cCREATE / INSERT — row mới được tạo
uUPDATE — row được cập nhật
dDELETE — row bị xóa
rREAD — từ initial snapshot
tTRUNCATE — table bị truncate
source — Metadata về nơi event xuất phát
"connector": "postgresql"
"db": "pdms_db"
"table": "documents"
"lsn": 24023424 ← WAL position, dùng làm idempotency key
"txId": 756 ← PostgreSQL transaction ID
"ts_ms": 1713168000000 ← Timestamp của transaction trong DB
"snapshot": "false"
